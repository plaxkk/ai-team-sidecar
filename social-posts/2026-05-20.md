# 开发手记 2026-05-20

今天主要把「做完一轮就忘」这件事往前推了一步。

以前 vibe coding 爽是爽，过一周翻 git log 也说不清自己到底干了啥。所以把 checkpoint、changelog、还有对外发 build in public 的稿子，串成一条线了。

具体是这样：
- episode 结束会自动落一个检查点
- 能从检查点拉出结构化 changelog，也能导出 CHANGELOG.md
- 顺手加了小红书推文模板，不用每次现憋文案

实现上多了三张表：checkpoints、changelog_entries、social_posts。同一个 episode 不会重复建 checkpoint；change_type 会按 episode_type 映射成 feature / fix / refactor / chore 这类。CLI 和 Dashboard 都能触发，自动、手动都行。

改动量不大：新文件 4 个、改 3 个，核心逻辑大概 350 行，主要在 analysis、collector、dashboard 和 bin。

对我这种一个人写的人来说，价值不在「功能多」，而是每一轮都有痕迹。AI 写得快，记录跟不上，后面复盘基本等于瞎猜。现在至少能对着检查点回忆「那天我到底在折腾什么」。
