# QA 检查清单

## 功能测试

- [ ] 正常路径：核心功能是否按预期工作？
- [ ] 异常路径：输入错误数据时的处理是否正确？
- [ ] 边界情况：极端输入（空值、超长、特殊字符）是否处理？

## 安全检查

- [ ] 密码/敏感数据是否加密存储？
- [ ] 用户输入是否做了转义/过滤（防 XSS/注入）？
- [ ] API 是否有权限检查？

## 性能检查

- [ ] 是否有不必要的 N+1 查询？
- [ ] 大数据量场景是否考虑分页？
- [ ] 是否有明显的性能瓶颈？

## 风险等级标注

每条 QA 意见必须标注风险等级：

- **[高]**：可能导致数据丢失、安全漏洞、核心功能不可用
- **[中]**：可能导致功能降级、用户体验差、性能问题
- **[低]**：代码风格、文档缺失、非核心功能的小问题

## QA 红线

以下情况 QA 不得通过：

- ❌ QA 段落为占位内容（"无需审查"、"代码看起来没问题"）
- ❌ 未列出任何边界情况
- ❌ 未标注任何风险等级
- ❌ 未发现任何潜在问题
