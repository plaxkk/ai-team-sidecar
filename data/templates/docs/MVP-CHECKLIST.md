# MVP 检查清单

## 开始前

- [ ] 目标用户是谁？（具体到角色/场景）
- [ ] 痛点是什么？（用一句话描述）
- [ ] P0 范围是什么？（只做一件事）
- [ ] 不做什么？（至少列出 2 项排除）
- [ ] 成功指标是什么？（可量化的数字）

## 实现中

- [ ] 技术方案是否提供了至少 2 个选项？
- [ ] 单次改动不超过 5 个文件？
- [ ] 是否有反面意见 / 魔鬼代言人分析？
- [ ] 是否有用户证据支持？
- [ ] 是否评估了商业价值？

## 交付前

- [ ] QA 是否独立审查（非占位内容）？
- [ ] 是否有验证步骤（build/test/deploy）？
- [ ] Tech Lead 是否给出 Go/No-Go？
- [ ] 是否列出了剩余风险？
- [ ] 是否有下一步最小动作？

## 红线（违反即停止）

- ❌ 引入新框架但无法证明能提升 P0 指标
- ❌ 跨模块重构没有明确的用户价值
- ❌ 一次性改动超过 5 个文件且无说明
- ❌ 没有验证证据就宣布完成
- ❌ QA 审查是占位内容（"无需审查"）
